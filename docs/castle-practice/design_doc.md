# Design Doc: 将棋囲い練習用アプリ (MVP)

## 1. 概要

ユーザーが指定した将棋の「囲い」に対し、最短手順を習得させるためのWebアプリケーション。
GitHub Pagesによる静的ホスティングを前提とした、クリック操作ベースの軽量な学習ツール。

## 2. システム構成

- **フロントエンド**: React (Vite)
- **ロジック**: shogi.js（盤面制御・合法手判定）
- **デプロイ**: GitHub Pages（静的ホスティング）

## 3. UI/UX 仕様 (MVP)

操作は全て「クリック」で完結する。

- **駒の選択**: 自分の駒をクリックすると、その駒がハイライトされ、移動可能なマスが視覚的に表示される。
- **駒の移動**: 移動可能なマスをクリックすることで駒が移動する。
- **囲い選択**: 練習する囲い（矢倉・美濃など）をプルダウンで指定。
- **最短手順ナビ**: 選択した囲いまでの最小手数と、現在の状態からの推奨手順を提示。

## 4. 技術的ロジック

### 盤面操作
- shogi.js の `getMovesFrom(pos)` を呼び出し、合法手をUIに反映させる。

### 最短手数算出
- BFS（幅優先探索）アルゴリズムを採用。
- 初期局面と目標局面（囲い完成）を定義。
- shogi.js を活用し、盤面状態をノードとして最短パスを算出。

### 状態管理
- `selectedPiece`: 選択中の座標を保持。
- `board`: 現在の対局状態を保持。

## 5. 開発ロードマップ

| フェーズ | タスク |
|---------|--------|
| Phase 1 | Vite/React環境構築 + shogi.js 導入 |
| Phase 2 | クリックによる駒選択・移動ロジックの実装 |
| Phase 3 | BFSによる最短手順算出ロジックの実装 |
| Phase 4 | GitHub Pagesへのデプロイパイプライン構築 |

## 6. 将来の拡張性 (Ver 2.0以降の検討事項)

- **危険タイミングの可視化**: 囲い構築中における相手からの「急所（攻め筋）」のハイライト表示。
- **学習ログの可視化**: 過去の練習における最短手数からの乖離を統計的に表示。
- **難易度別モード**: 駒落ちや持ち駒制限など、実戦的な制約モードの追加。
